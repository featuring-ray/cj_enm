# CJ x 피처링 온스타일 크리에이터 플랫폼

## 프로젝트 개요

CJ 온스타일의 크리에이터 커머스 생태계를 위한 4개 플랫폼 목업 구현 프로젝트.
모든 데이터는 Mock 데이터를 사용하며, 실제 API 연동 없이 프론트엔드 목업으로 구현한다.

### 플랫폼 구성

| 플랫폼 | 코드 | 대상 | 설명 |
|--------|------|------|------|
| **온트너(Ontner)** | O | 크리에이터(인플루언서) | 크리에이터가 캠페인을 탐색하고 성과를 확인하는 서비스 |
| **온트러스트(Ontrust)** | T | CJ 내부 MD | MD가 크리에이터를 탐색·추천·관리하고 캠페인 성과를 분석하는 내부 도구 |
| **파트너(Partner)** | P | 외부 협력사/브랜드 | 협력사가 온트너 등록 크리에이터만 제한적으로 조회하는 서비스 |
| **어드민(Admin)** | A | 시스템 관리자 | DM 발송 정책, 계정 관리 등 시스템 관리 도구 |

### IA (정보구조) ID 체계

화면 ID는 `{플랫폼}-{도메인}-{번호}` 형식을 따른다.

| 플랫폼 코드 | 도메인 코드 | 의미 |
|-------------|-------------|------|
| O (온트너) | A: 회원, B: 마이페이지, C: 캠페인, D: 성과 | 크리에이터 기능 |
| T (온트러스트) | A: 탐색/DM, B: 추천, C: 성과/리포트, D: 설정 | MD 기능 |
| P (파트너) | C: 성과/리포트 | 협력사 기능 |
| A (어드민) | A: 관리 | 시스템 관리 |

### 4대 기능 도메인

| 도메인 | 화면 ID 범위 | 설명 |
|--------|-------------|------|
| **회원관리** | O-A-01, O-B-01~03, T-D-01 | 가입, 프로필, 계정 연계 |
| **캠페인 탐색 관리** | T-A-01~02, T-A-07~14, T-B-01, O-A-10, O-C-03~07 | 크리에이터/캠페인/콘텐츠 탐색, 추천, 북마크 |
| **성과 관리** | O-D-01~03, T-C-01~06, P-C-01 | 성과 조회, 인사이트 리포트 |
| **DM 관리** | T-A-03~06, T-A-05-1, A-A-01 | DM 발송, 템플릿, 어드민 |

### 핵심 용어

- **크리에이터**: 인스타그램/유튜브 인플루언서 (CJ 온스타일과 협업)
- **캠페인**: CJ가 운영하는 마케팅/판매 캠페인 (공동구매, 브랜드 협업 등)
- **어필리에이트/리워드 링크**: 크리에이터가 공유하는 성과 기반 수익 링크
- **공구 콘텐츠**: 공동구매 관련 인스타그램/유튜브 콘텐츠
- **크루찾기**: 팔로워 유사도 분석 기능 (베타)
- **인게이지먼트**: 좋아요, 댓글, 저장, 공유 등 콘텐츠 반응 지표
- **인게이지먼트 가중치**: 댓글 > 공유 > 저장/조회수 > 좋아요 > 팔로워 순 가중 점수
- **온트너 회원**: 온트너 플랫폼에 가입한 크리에이터 (전체 인플루언서의 일부)
- **배치 추천**: 1일 1회 주기로 사전 계산된 추천 결과 (실시간 연산 아님)
- **Fallback 로직**: 1순위 데이터 없을 시 차선 기준으로 순차 탐색하는 방식
- **D+14 확정 매출**: 캠페인 종료 후 14일 기준 취소/교환/반품 반영 최종 매출

## 기술 스택

- **프레임워크**: Next.js (App Router)
- **언어**: TypeScript
- **스타일링**: Tailwind CSS
- **컴포넌트**: shadcn/ui
- **차트**: Recharts
- **상태관리**: React Context (또는 Zustand)
- **Mock 데이터**: JSON 파일 또는 인메모리
- **아이콘**: Lucide React

### 2. 캠페인 탐색 관리

#### 2-1. 크리에이터 탐색/추천

##### T-A-01: 크리에이터 탐색 `/ontrust/creator/search`
> **플랫폼**: 온트러스트 | **유형**: Page | **R&R**: CJ+피처링 협업

**요구사항**: R010 | **상태**: 결정 필요

**주요 기능:**
1. **탐색**
   - 피처링 전체 크리에이터 풀 대상 검색 (파트너는 온트너 회원만)
   - 검색필터: 브랜드 키워드 검색, 플랫폼 선택 (인스타그램/유튜브)
   - 캠페인 선택 (캠페인 기반 탐색 시)
   - 목록에 온트너 회원 여부 뱃지 표시

2. **검색 결과 정렬**
   - 인게이지먼트 가중치 기반 점수화 정렬:
     - 댓글 > 공유 > 저장/조회수 > 좋아요 > 팔로워 순
   - 최근 3개월 기준

3. **검색 결과 테이블**
   - 프로필 이미지, 핸들명, 팔로워 수, 인게이지먼트율
   - 온트너 회원 여부 뱃지
   - 세일즈 단가 컬럼 (데이터 있는 경우만)
   - CJ 플랫폼 내 캠페인 진행 이력 (누적 캠페인 수)
   - 카테고리

4. **필터 패널:**
   - 피처링 기존 고급 필터 모두 포함
   - 카테고리 포함/제외 기능
   - ~~브랜드 필터~~ → 키워드 검색으로 대체 (별도 '브랜드 검색' UI 제공)
   - ~~공동구매 계정~~ → '공동구매 진행 계정만 보기' 버튼/필터 (키워드 검색 기반 동작)
   - ~~세일즈 단가 필터~~ → **불가** (결과 테이블 컬럼 표기만 가능)
   - ~~온트너 회원만 보기/제외~~ → **검색 필터 불가**, 결과 테이블에 뱃지로만 표시
   - 특정 계정 제외 (브랜드 공식계정, 연예인 계정 - 사전 정의 리스트)

5. **행 액션:**
   - "상세 보기" → T-A-02
   - "캠페인 참여 제안" (온트너 회원인 경우에만)
   - "온트너 회원가입 제안" (DM/이메일)
   - "캠페인참여 + 회원가입 제안" (DM/이메일)
   - "북마크" → T-A-10 그룹에 추가
   - "N명의 크리에이터 선택 후 팔로워 유사도 분석" → T-A-07

6. **브랜드 속성 추출**
   - 크리에이터의 콘텐츠에서 브랜드 속성 추출 (로직 필요)

**데이터 소스:**
- CJ: 회원정보, 캠페인 이력
- 피처링: 크리에이터 분석 데이터, 인게이지먼트 지표

**제약사항:**
- 피처링 검색 엔진 구조상 외부 시스템(CJ) 데이터 기반 필터를 직접 검색 조건으로 사용 불가
- 브랜드 필터는 정확한 매칭이 아닌 콘텐츠 내 브랜드 언급 기반 탐색
- 다른 채널(X, 틱톡, 네이버블로그) 탐색 시 피처링 화면으로 이동 안내문구 추가

##### T-A-02: 크리에이터 상세 리포트 `/ontrust/creator/[id]`
> **플랫폼**: 온트러스트 | **유형**: Page | **R&R**: CJ+피처링 협업

- 크리에이터 프로필 헤더 (온트너 회원정보 + 피처링 정보 통합)
- 오디언스 분석 섹션:
  - 성별/연령 분포 (바 차트)
  - 지역 분포
  - 참고 텍스트: "콘텐츠 반응 기준 예측 데이터 (실제 팔로워 기준과 다를 수 있음)"

## 주의사항

- 모든 데이터는 Mock이다. API 호출 시뮬레이션은 `setTimeout`으로 딜레이만 넣는다.
- "불가" 표시된 필터(세일즈 단가, 온트너 회원 필터링)는 UI에 넣되, 비활성화 상태 + 툴팁으로 "데이터 미제공" 안내를 표시한다.
- 추천 알고리즘 결과는 Mock 데이터에서 미리 계산된 값으로 표시한다. 실제 ML 로직은 구현하지 않는다.
- 크루찾기(팔로워 유사도)의 "분석 중" 상태를 반드시 구현한다 (배치 처리 시뮬레이션).
- 인사이트 리포트에서 AI 텍스트 생성은 제외한다. 고정 텍스트 템플릿에 숫자만 동적으로 바인딩한다.
- 콘텐츠 트래킹 데이터는 시간대별 Mock 데이터를 생성하여 차트로 표시한다.
- 댓글 키워드 워드 클라우드는 Mock 키워드 빈도 데이터로 구현한다.
- 온트너/온트러스트/파트너/어드민은 같은 앱 내 라우팅으로 구분하되, 각각 독립적인 레이아웃을 가진다.
- 캠페인 상세(O-C-04)의 연관 콘텐츠 Fallback 로직을 반드시 구현한다 (상품→브랜드→소분류→숨김).
- 캠페인별 성과 조회에서 D+14 확정 매출 상태를 표시한다 (진행중 vs 확정 구분 UI).
- 캠페인 추천 모듈(O-C-06)은 쇼핑몰식 가로 롤링 UI로 구현한다 (3테마 각각 독립 캐러셀).
- 파트너 플랫폼은 온트너 회원 크리에이터만 검색 가능하도록 데이터 필터링을 적용한다.
- 브랜드 역제안(O-C-07)은 피처링 API 응답 Mock으로만 구현하고 CJ UI 영역은 기본 레이아웃만 구현한다.
