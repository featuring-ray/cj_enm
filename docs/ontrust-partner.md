## 1. 캠페인 탐색 관리

### 1-1. 크리에이터 탐색/추천

#### T-A-01: 크리에이터 탐색 `/ontrust/creator/search`
> **유형**: Page | **R&R**: CJ+피처링 협업

**요구사항**: R010 | **상태**: 결정 필요

**주요 기능:**
1. **탐색**
   - 피처링 전체 크리에이터 풀 대상 검색 (파트너는 온트너 회원만)
   - 검색필터: 브랜드 키워드 검색, 플랫폼 선택 (인스타그램/유튜브)
   - 캠페인 선택 (캠페인 기반 탐색 시)
   - 목록에 온트너 회원 여부 뱃지 표시

2. **검색 결과 정렬**
   - 인게이지먼트 가중치 기반 점수화 정렬:
     - 댓글 > 공유 > 저장/조회수 > 좋아요 > 팔로워 순
   - 최근 3개월 기준

3. **검색 결과 테이블**
   - 프로필 이미지, 핸들명, 팔로워 수, 인게이지먼트율
   - 온트너 회원 여부 뱃지 (O/X)
   - 세일즈 단가 레인지 컬럼 (데이터 있는 경우만)
   - CJ 플랫폼 내 캠페인 진행 이력 (온트너 누적 캠페인 수)
   - 카테고리
   - **컨택 가능 이메일** (크리에이터 프로필에서 파싱한 공식 이메일 주소)
   - **대댓글 자동화 툴 사용 여부** (소셜비즈 등) — 평균 댓글 수 옆에 아이콘/뱃지로 표기

4. **필터 패널:**
   - 피처링 기존 고급 필터 모두 포함
   - 카테고리 포함/제외 기능
   - ~~브랜드 필터~~ → 키워드 검색으로 대체 (별도 '브랜드 검색' UI 제공)
   - ~~공동구매 계정~~ → '공동구매 진행 계정만 보기' 버튼/필터 (키워드 검색 기반 동작)
   - ~~세일즈 단가 필터~~ → **불가** (결과 테이블 컬럼 표기만 가능)
   - ~~온트너 회원만 보기/제외~~ → **검색 필터 불가**, 결과 테이블에 뱃지로만 표시
   - 특정 계정 제외 (브랜드 공식계정, 연예인 계정 - 사전 정의 리스트)

5. **행 액션:**
   - "상세 보기" → T-A-02
   - "캠페인 참여 제안" (온트너 회원인 경우에만)
   - "온트너 회원가입 제안" (DM/이메일)
   - "캠페인참여 + 회원가입 제안" (DM/이메일)
   - "북마크" → T-A-10 그룹에 추가
   - "N명의 크리에이터 선택 후 팔로워 유사도 분석" → T-A-07

6. **브랜드 속성 추출**
   - 크리에이터의 콘텐츠에서 브랜드 속성 추출 (로직 필요)

**데이터 소스:**
- CJ: 회원정보, 캠페인 이력
- 피처링: 크리에이터 분석 데이터, 인게이지먼트 지표

**제약사항:**
- 피처링 검색 엔진 구조상 외부 시스템(CJ) 데이터 기반 필터를 직접 검색 조건으로 사용 불가
- 브랜드 필터는 정확한 매칭이 아닌 콘텐츠 내 브랜드 언급 기반 탐색
- 다른 채널(X, 틱톡, 네이버블로그) 탐색 시 피처링 화면으로 이동 안내문구 추가

#### T-A-09: 콘텐츠 탐색 → 크리에이터 추천 `/ontrust/creator/content`
> **유형**: Page | **R&R**: 피처링+CJ 협업

**상태**: 결정 필요

**주요 기능:**
- **인게이지먼트 상위 공구 콘텐츠 Top 100 랭킹**
  - 인게이지먼트 가중치 점수 기반 정렬 (댓글 5.0 > 공유 4.0 > 저장 3.0 > 조회수 2.0 > 좋아요 1.5 > 팔로워 0.01)
  - 카테고리별, 기간별(1주일, 1개월, 3개월, 6개월) 조회
  - **대댓글 자동화 툴 사용 여부** 결과 리스트 내 표기
- **연관 크리에이터 정보 매핑**
  - 각 콘텐츠 옆에 해당 콘텐츠를 제작한 크리에이터의 핵심 정보 표시
  - 연관 캠페인(리워드 링크) 정보가 있다면 매핑하여 탐색 가능하게 함
  - 해당 크리에이터의 온트너 회원 여부 뱃지 표시
- 콘텐츠 선택 시: 해당 크리에이터 정보 사이드 패널
- 크리에이터 상세(T-A-02)로 이동 링크
- **필터: T-A-01과 동일한 고급 필터 적용**
  - 카테고리 포함/제외
  - 브랜드 일치/유사 브랜드군 포함 (키워드 검색으로 동작)
  - 공동구매 진행 콘텐츠만 보기 (키워드 검색 기반)
  - 특정 계정 제외 (브랜드 공식계정, 연예인 계정 — 사전 정의 리스트)

**콘텐츠 컬럼 표기:**
- 기본 지표: 순위뱃지, 원본 SNS썸네일, SNS플랫폼, 주요 성과 지표
- 연관 정보: 크리에이터 핵심 정보, 캠페인/리워드 링크 매핑

**미결사항:**
- 콘텐츠 카테고리 분류: 피처링에서 콘텐츠 개별 카테고리를 적재하고 있지 않음 (크리에이터 단위 카테고리만 보유)
- 추가 비용 발생 가능

**데이터 소스:**
- 피처링: 콘텐츠 크롤링, 인게이지먼트 데이터

#### T-B-01: 크리에이터 추천 `/ontrust/creator/recommend`
> **유형**: Page + Module | **R&R**: 피처링 단독

**상태**: 확정

**주요 기능:**
1. **Top 10 리스팅 뷰**
   - 캠페인 선택 드롭다운
   - 캠페인 정보(카테고리, 브랜드) 기반 최적 크리에이터 Top 10 자동 추천
   - 추천 점수 표시: 인게이지먼트 60% + 매출 실적 40% 가중
   - CJ 캠페인 진행 이력 + 매출 데이터가 있는 크리에이터 가중치 부여
   - "배치 업데이트: YYYY-MM-DD" 표시
   - 각 카드에서 바로 "제안하기" 가능

2. **더보기 (탐색 랜딩)**
   - "더보기" 클릭 시 추천에 사용된 조건(필터)이 세팅된 상태로 T-A-01 크리에이터 탐색으로 이동

3. **노출 영역 (진입점)**
   - 독립 페이지: `/ontrust/creator/recommend`
   - 캠페인 등록/조회 화면 내 추천 영역 (Module로 임베드)
   - 인사이트 리포트(브랜드 기준) 하단 추천 영역

4. **파트너 제한사항**
   - 파트너 계정은 온트너 회원 크리에이터 풀 안에서만 추천 결과 제공

**추천 알고리즘:**
- 매핑 기준: 카테고리 일치도, 브랜드 유사도, 공구 진행 여부
- **랭킹 산정 가중치:**
  - CJ와 캠페인을 진행하여 '매출 데이터' 실적이 있는 크리에이터: 인게이지먼트 60% + 매출 실적 40%
  - 매출 데이터 없는 크리에이터: 인게이지먼트 100%
  - 인게이지먼트 점수는 공통 가중치 공식 적용 (댓글 5.0 > 공유 4.0 > 저장 3.0 > 조회수 2.0 > 좋아요 1.5)
- **데이터 처리: 배치 기반 (1일 1회 업데이트)**
  - 미리 조건에 맞는 풀을 계산해 두는 배치 방식으로 동작 (시스템 과부하 방지)
  - '크리에이터 탐색' 화면 수치와 '추천' 화면 수치가 시간차로 다를 수 있음
  - 배치 업데이트 시각 UI에 표시, 24시간+ 경과 시 경고 배너

**데이터 소스:**
- CJ: 캠페인/매출 데이터
- 피처링: 인게이지먼트 데이터

**미결사항:**
- 추천-탐색 메뉴 구조: [옵션A] 별도 메뉴 vs [옵션B] 탐색 내 추천 탭
- 가중치 비율 최종 확정 필요

#### T-A-15: 컨택 요청 관리 `/ontrust/campaign/contact-requests`
> **유형**: Page | **R&R**: CJ+피처링 협업

**요구사항**: 크리에이터 컨택 요청 | **상태**: NEW

**목적 (User Story):**
MD는 원하는 크리에이터를 찾고 내부 운영 부서(파트너 사업부 등)로 공식 컨택 요청을 보내 직접 연락하는 실무적 부담을 해소할 수 있다.

---

**■ 컨택 요청 발송 (MD → 내부 운영팀)**

MD는 피처링 연동 탐색 뷰(T-A-01)에서 원하는 크리에이터를 선택한 뒤, 직접 DM 대신 CJ 내부 '파트너사업부'팀에 '컨택 요청'을 발송할 수 있다.
- 발송된 컨택 요청 건들은 온트러스트 시스템 내에 목록으로 축적된다.

**■ 컨택 이메일 노출**

크리에이터 탐색 결과 화면(T-A-01)에 크리에이터가 프로필에 기재한 '컨택 가능한 공식 이메일 주소'를 파싱하여 결과값으로 노출
- 인스타그램 bio 내 이메일 패턴 파싱 (정규식 기반)
- 결과 테이블 컬럼에 '컨택 이메일'로 노출

**■ 스펙아웃**

- 온트러스트 계정마다 피처링을 연동하여 피처링 솔루션 내에서 DM을 발송하는 기능은 **제외**
- 실제 크리에이터와의 연락은 담당자들이 기존 방식(개별 핸드폰 등)으로 진행

---

**컨택 요청 워크플로우:**
1. MD가 T-A-01에서 크리에이터 탐색 → 원하는 크리에이터 선택
2. "컨택 요청" 발송 (요청 사유/메모 입력)
3. 요청이 T-A-15 목록에 적재
4. 내부 운영 부서 담당자가 상태값으로 처리 상황을 일괄 관리

**주요 기능:**

1. **컨택 요청 발송** (T-A-01 탐색 뷰 연동)

   - 요청 데이터: 요청자(MD), 대상 크리에이터, 요청일시, 요청 사유
   - 발송 시 온트러스트 DB에 즉시 저장

2. **컨택 요청 관리** (내부 운영 부서)
   - MD들이 발송한 컨택 요청 건이 온트러스트 시스템 내 목록으로 축적
   - 내부 담당자가 실제 컨택 진행 여부에 대한 상태값을 일괄 관리

3. **상태 관리 테이블**
   - 컬럼: `요청ID` | `요청자` | `크리에이터` | `요청일` | `상태` | `처리자` | `처리일` | `비고`
   - 필터: 상태별 (접수/검토 중/컨택 진행/완료/미진행), 기간별, 요청자별

   - 페이지네이션 적용

4. **상태 전이**
   - `접수`(기본) → `검토 중` → `컨택 진행` / `미진행`
   - `컨택 진행` → `완료`
   - 내부 담당자가 드롭다운으로 상태 변경 (일괄 변경 지원)

**데이터 저장:**

  - 요청자(MD), 대상 크리에이터, 요청일시, 요청 사유, 상태값

**파트 1. 통합 Summary**

캠페인 기간 동안 콘텐츠 업로드 추이, 콘텐츠 인게이지먼트 추이, 주문 수량 추이를 하나의 그래프로 통합 표시. 콘텐츠 인게이지먼트 수치와 실제 매출 수치를 오버레이하여 연관성, 추이를 직관적으로 파악.

- **그래프 유형**: 복합 차트 (라인 + 바)
  - X축: 시간 (시간별/일별 선택 가능)
  - Y축(좌): 인게이지먼트 수치 (좋아요, 댓글, 조회수)
  - Y축(우): 주문 수량 / 매출액
  - 콘텐츠 업로드 시점: 마커로 표시

**파트 2. 콘텐츠 기준 상세 (피처링 데이터)**

- 콘텐츠 업로드 시간/일자
- 콘텐츠 유형 (인스타: 피드/릴스, 유튜브: 숏폼/롱폼)
- 콘텐츠별 인게이지먼트: 좋아요, 댓글, 조회수
- 주요 반응 키워드
- 시간/일자별 콘텐츠 업로드 현황
- **결측치 보정 정책:** 네트워크 오류 등으로 특정 시간대 수집 실패 시 빈 구간이 발생할 수 있으며, 직전 데이터 유지 방식으로 보정 처리 + "보정된 데이터" 툴팁 표시

- **데이터 소스**: 피처링 콘텐츠 트래킹 수집 결과
  → 캠페인 기간 내 크리에이터 계정 콘텐츠 전량 수집
  → 캠페인 연관도 점수 기반 필터링 옵션 제공
- **테이블 컬럼**: 업로드일시 | 콘텐츠유형 | 썸네일 | 좋아요 | 댓글 | 조회수 | 연관도 | 주요키워드

**파트 3. 매출 기준 상세 (CJ 데이터)**

- 시간별 주문 수량 (전체, 옵션별) 및 매출 현황
- 주문 고객 세그먼트: 연령, 성별
- 함께 구매한 상품 Top 10
  → 연관 상품 로직 적용
  → 최소 주문 고객 10명 이상, 주문고객수 → 금액순 정렬
- 취소 / 교환 / 반품 현황:
  → 취교반율, 취교반 주요 사유
  → 캠페인 종료 후 14일까지 데이터 미확정 표시
  → 14일 경과 후 확정 뱃지 전환
  → 주요 사유: CJ CS 데이터에서 사유 코드별 집계
- **D+14 확정 매출:** 캠페인 종료 후 14일 기준 최종 매출 확정 (취교반 수치 업데이트 소요 시간 약 14일)

### 3-2. 캠페인 인사이트 리포트

> **목적 (User Story):** MD는 상황과 목적에 맞춰 특정 크리에이터의 전반적 활동 성과를 묶어서 보거나, 특정 브랜드의 캠페인 종합 성과를 묶어서 분석할 수 있다.

#### 개요

특정 기간 및 조건(크리에이터/브랜드)을 설정하여, 온트너보다 더 넓은 범위의 다각도 성과 지표를 분석할 수 있는 조회형 리포트.

- **집계 대상**: 설정 기간 내 '진행 완료'된 캠페인 기준 (진행 중 캠페인 제외)
- **리포트 형태**: 기간 설정 + 조건 선택 조회형

#### 온트너와의 핵심 차이점

| 항목 | 온트너 | 온트러스트 |
|------|--------|-----------|
| 조회 축 | 자기 것만 | 크리에이터 기준 / 브랜드 기준 선택 |
| 크리에이터 기준 | 캠페인 + 리워드링크 탭 (내 캠페인만) | 캠페인 + 리워드링크 탭 (전체 캠페인) |
| 브랜드 기준 | — | 캠페인 리포트만 (리워드링크 탭 미노출) |
| 지표 범위 | 기본 | 넓은 범위 (Top 20, 협업 크리에이터 분석 등) |

- **리포트 엔진**: 피처링 API (인게이지먼트 데이터) + CJ API (매출, 구매 데이터) 병합
- **집계 쿼리**: `WHERE 캠페인상태 = '완료' AND 기간 BETWEEN 시작일 AND 종료일`
- **조회 축 분기**:
  → 크리에이터 기준: `creatorId` 파라미터 → 해당 크리에이터의 전체 캠페인 조회
  → 브랜드 기준: `brandId` 파라미터 → 해당 브랜드의 전체 캠페인 조회
- 최대 조회 기간 제한 필요 (DB 부하 방지)

#### 리포트 조회 프로세스

**Step 1 (기간 설정)**:
- 달력에서 시작일~종료일 자유 설정
- 최대 조회 가능 기간 제한 적용 (DB 부하 방지)
- UI: `DatePicker` → `startDate`, `endDate` 파라미터

**Step 2 (조회 축 선택)**:
- '크리에이터 단위' 조회 vs '브랜드 단위' 조회
- UI: 라디오 버튼 → `viewType = 'creator' | 'brand'`

**Step 3-A (크리에이터 기준)**:
- 특정 크리에이터 검색 → 해당 기간 내 진행한 캠페인 리스트(단일/다중) 선택 후 조회
- [결과] 캠페인 탭 + 리워드 링크 탭 제공
- 온트너 리포트와 동일 지표 + 전체 캠페인 내역 열람
- UI: 크리에이터 검색 API → 결과 선택 → 캠페인 리스트 API (해당 크리에이터 + 기간 필터) → 단일/다중 선택 → 리포트 생성

**Step 3-B (브랜드 기준)**:
- 특정 브랜드 검색 → 해당 브랜드 캠페인 리스트(단일/다중) 선택 후 조회
- [결과] 캠페인 리포트만 제공 (리워드 링크 탭 미노출)
- UI: 브랜드 검색 API → 결과 선택 → 캠페인 리스트 API (해당 브랜드 + 기간 필터) → 리포트 생성 (리워드링크 탭 비활성)

#### T-C-03: 인사이트 리포트 - 크리에이터 기준 `/ontrust/insight`
> **유형**: Page | **R&R**: 피처링+CJ 협업

**요구사항**: R015 | **상태**: 확정

**[원칙]**

**[추가 제공]**

5. **캠페인 구매 분석**

   - b) 고객 리텐션: 재구매 여부 및 리텐션율
   - c) 판매 상품 리뷰 연관 키워드 요약
   - d) 오디언스 대비 구매전환수 인사이트

6. **리포트 하단 추천 모듈**

**데이터 소스:**

## UI/UX 가이드라인

> 참고: 실제 온트러스트는 **웹스퀘어(CJ 내부 프레임워크)** 기반이지만, 본 목업에서는 shadcn/ui로 대체 구현한다. 아래 규칙은 실제 온트러스트 QA 화면의 디자인 패턴을 분석하여 정리한 것이다.

### 1. 디자인 원칙

- **플랫 디자인**: 모든 요소의 border-radius 0px (직각 모서리), box-shadow 없음
- **고밀도 레이아웃**: 컴팩트한 입력/버튼/행 사이즈로 한 화면에 많은 정보를 표시
- **전면 보더**: 테이블, 폼 셀 등 모든 영역에 1px solid 보더 적용
- **보라색 포인트**: 주요 액션과 브랜딩에 보라색(#7c3aed) 일관 사용

### 2. 레이아웃 구조

#### 2-1. GNB (Global Navigation Bar) — 상단 고정, 2줄

| 줄 | 내용 | 높이 |

### 6. 컴포넌트 규칙

- 보더: `2px solid #7c3aed` (보라색 강조)
- 배경: 흰색
- 패딩: 12px
- "조회" 버튼: `otr-btn-primary` (보라 배경)
- "초기화" 버튼: `otr-btn-secondary` (흰 배경)
- 상태 드롭다운 옵션 예: `-전체-`, `요청완료`, `반려`, `승인완료`

| 열 수 | CSS 클래스 | grid-template-columns |

| 3열 | `otr-form-grid-3col` | `120px 1fr 120px 1fr 120px 1fr` |
| 4열 | `otr-form-grid-4col` | `100px 1fr 100px 1fr 100px 1fr 100px 1fr` |

- **라벨 셀** (`otr-form-label`): 회색 배경(#f5f5f5), 12px, 우측 정렬, 1px 보더
- **값 셀** (`otr-form-value`): 흰색 배경, 13px, 좌측 정렬, 1px 보더
- **필수 표시** (`otr-required`): 라벨 뒤에 보라색 `■` 마커
- **값 병합**: `span-2` (3칸), `span-3` (5칸), `full` (나머지 전부)

◆ 진행 정보

- `◆` 다이아몬드: 보라색 (#7c3aed)
- 제목: 13px, bold, #333333
- 마커와 제목 사이 4px 간격

#### 6-4. 데이터 테이블 (`otr-data-table`)

| 속성 | 규칙 |
|------|------|
| 보더 | 테이블 전체 + 모든 셀 1px solid #e0e0e0 |

| 행 높이 | 34px |
| 행 호버 | 없음 (기본) 또는 연한 회색 |
| 선택 행 | 연한 보라 배경 (#f3f0ff) |

- **첫 번째 컬럼**: 체크박스 (일괄 선택/개별 선택)
- **두 번째 컬럼**: No (행 번호, 1부터 시작)
- 빈 데이터: 가운데 정렬, "조건에 맞는 데이터가 없습니다." + 아이콘

#### 6-5. 버튼 3종

| 종류 | 클래스 | 배경 | 텍스트 | 보더 | 용도 |

- 패딩: 1px 6px

#### 6-12. 다이얼로그/모달

- 헤더: 제목 + 설명
- 본문: space-y-4 간격
- 푸터: 좌측 취소(Secondary) + 우측 확인(Primary)

### 8. 인터랙션 규칙

| 인터랙션 | 동작 |
|----------|------|
| 행 선택 | 체크박스 클릭 → 연보라 배경(#f3f0ff) 하이라이트 |
| 행 호버 액션 | 호버 시 우측에 `⋯` 더보기 메뉴 표시 (opacity 전환) |
| 비활성 메뉴 | 회색 텍스트 + `cursor: not-allowed` + 클릭 불가 |
| 필수 필드 | 라벨 뒤에 보라색 `■` 마커 |
| 비활성 필터 | UI에 표시하되 비활성 상태 + 툴팁 "데이터 미제공" |
| 드롭다운 상태 | 전체/요청완료/반려/승인완료 등 상태 옵션 |
| 일괄 액션 | N건 선택 시 툴바에 일괄 액션 버튼 활성화 |
| 링크 클릭 | 텍스트 밑줄 + 보라색, 클릭 시 상세 페이지 이동 |

### 9. CJ 자체 영역 vs 피처링 구현 영역

- **CJ 자체 메뉴** (disabled): LNB에 표시하되 클릭 불가, 회색 처리
  - 예: 온트너 계정 관리, 캠페인 리스트, 정산현황, 전자계약, 고객, 상품 일부
- **피처링 구현 영역**: 활성 상태, 클릭 가능, 보라색 하이라이트
  - 예: 크리에이터 탐색, 인사이트 리포트, DM 관리, 성과 조회
- CJ 자체 영역의 페이지 내용은 구현하지 않되, 사이드바 메뉴에는 표시하여 실제 시스템의 전체 구조를 보여준다
