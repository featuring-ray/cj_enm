## 1. 캠페인 탐색 관리

### 1-1. 크리에이터 탐색/추천

#### T-A-01: 크리에이터 탐색 `/ontrust/creator/search`
> **유형**: Page | **R&R**: CJ+피처링 협업

**요구사항**: R010 | **상태**: 결정 필요

**주요 기능:**
1. **탐색**
   - 피처링 전체 크리에이터 풀 대상 검색 (파트너는 온트너 회원만)
   - 검색필터: 브랜드 키워드 검색, 플랫폼 선택 (인스타그램/유튜브)
   - 캠페인 선택 (캠페인 기반 탐색 시)
   - 목록에 온트너 회원 여부 뱃지 표시

2. **검색 결과 정렬**
   - 인게이지먼트 가중치 기반 점수화 정렬:
     - 댓글 > 공유 > 저장/조회수 > 좋아요 > 팔로워 순
   - 최근 3개월 기준

3. **검색 결과 테이블**
   - 프로필 이미지, 핸들명, 팔로워 수, 인게이지먼트율
   - 온트너 회원 여부 뱃지 (O/X)
   - 세일즈 단가 레인지 컬럼 (데이터 있는 경우만)
   - CJ 플랫폼 내 캠페인 진행 이력 (온트너 누적 캠페인 수)
   - 카테고리
   - **컨택 가능 이메일** (크리에이터 프로필에서 파싱한 공식 이메일 주소)
   - **대댓글 자동화 툴 사용 여부** (소셜비즈 등) — 평균 댓글 수 옆에 아이콘/뱃지로 표기

4. **필터 패널:**
   - 피처링 기존 고급 필터 모두 포함
   - 카테고리 포함/제외 기능
   - ~~브랜드 필터~~ → 키워드 검색으로 대체 (별도 '브랜드 검색' UI 제공)
   - ~~공동구매 계정~~ → '공동구매 진행 계정만 보기' 버튼/필터 (키워드 검색 기반 동작)
   - ~~세일즈 단가 필터~~ → **불가** (결과 테이블 컬럼 표기만 가능)
   - ~~온트너 회원만 보기/제외~~ → **검색 필터 불가**, 결과 테이블에 뱃지로만 표시
   - 특정 계정 제외 (브랜드 공식계정, 연예인 계정 - 사전 정의 리스트)

5. **행 액션:**
   - "상세 보기" → T-A-02
   - "캠페인 참여 제안" (온트너 회원인 경우에만)
   - "온트너 회원가입 제안" (DM/이메일)
   - "캠페인참여 + 회원가입 제안" (DM/이메일)
   - "북마크" → T-A-10 그룹에 추가
   - "N명의 크리에이터 선택 후 팔로워 유사도 분석" → T-A-07

6. **브랜드 속성 추출**
   - 크리에이터의 콘텐츠에서 브랜드 속성 추출 (로직 필요)

**데이터 소스:**
- CJ: 회원정보, 캠페인 이력
- 피처링: 크리에이터 분석 데이터, 인게이지먼트 지표

**제약사항:**
- 피처링 검색 엔진 구조상 외부 시스템(CJ) 데이터 기반 필터를 직접 검색 조건으로 사용 불가
- 브랜드 필터는 정확한 매칭이 아닌 콘텐츠 내 브랜드 언급 기반 탐색
- 다른 채널(X, 틱톡, 네이버블로그) 탐색 시 피처링 화면으로 이동 안내문구 추가

#### T-A-02: 크리에이터 상세 리포트 `/ontrust/creator/[id]`
> **유형**: Page | **R&R**: CJ+피처링 협업

- 크리에이터 프로필 헤더 (온트너 회원정보 + 피처링 정보 통합)
- 오디언스 분석 섹션:
  - 성별/연령 분포 (바 차트)

  - 참고 텍스트: "콘텐츠 반응 기준 예측 데이터 (실제 팔로워 기준과 다를 수 있음)"
- 콘텐츠 인게이지먼트 섹션:
  - 최근 콘텐츠 리스트 + 인게이지먼트 지표
  - 평균 인게이지먼트율 추이 (라인 차트)
- 캠페인 이력 (온트너 공개 캠페인)
- 하단 액션: "캠페인 제안", "북마크에 추가", "DM 발송"

#### T-A-09: 콘텐츠 탐색 → 크리에이터 추천 `/ontrust/creator/content`
> **유형**: Page | **R&R**: 피처링+CJ 협업

**상태**: 결정 필요

**주요 기능:**
- **인게이지먼트 상위 공구 콘텐츠 Top 100 랭킹**
  - 인게이지먼트 가중치 점수 기반 정렬 (댓글 5.0 > 공유 4.0 > 저장 3.0 > 조회수 2.0 > 좋아요 1.5 > 팔로워 0.01)
  - 카테고리별, 기간별(1주일, 1개월, 3개월, 6개월) 조회
  - **대댓글 자동화 툴 사용 여부** 결과 리스트 내 표기
- **연관 크리에이터 정보 매핑**
  - 각 콘텐츠 옆에 해당 콘텐츠를 제작한 크리에이터의 핵심 정보 표시
  - 연관 캠페인(리워드 링크) 정보가 있다면 매핑하여 탐색 가능하게 함
  - 해당 크리에이터의 온트너 회원 여부 뱃지 표시
- 콘텐츠 선택 시: 해당 크리에이터 정보 사이드 패널
- 크리에이터 상세(T-A-02)로 이동 링크
- **필터: T-A-01과 동일한 고급 필터 적용**
  - 카테고리 포함/제외
  - 브랜드 일치/유사 브랜드군 포함 (키워드 검색으로 동작)
  - 공동구매 진행 콘텐츠만 보기 (키워드 검색 기반)
  - 특정 계정 제외 (브랜드 공식계정, 연예인 계정 — 사전 정의 리스트)

**콘텐츠 컬럼 표기:**
- 기본 지표: 순위뱃지, 원본 SNS썸네일, SNS플랫폼, 주요 성과 지표
- 연관 정보: 크리에이터 핵심 정보, 캠페인/리워드 링크 매핑

**미결사항:**
- 콘텐츠 카테고리 분류: 피처링에서 콘텐츠 개별 카테고리를 적재하고 있지 않음 (크리에이터 단위 카테고리만 보유)
- 추가 비용 발생 가능

**데이터 소스:**
- 피처링: 콘텐츠 크롤링, 인게이지먼트 데이터

#### T-B-01: 크리에이터 추천 `/ontrust/creator/recommend`
> **유형**: Page + Module | **R&R**: 피처링 단독

**상태**: 확정

**주요 기능:**
1. **Top 10 리스팅 뷰**
   - 캠페인 선택 드롭다운
   - 캠페인 정보(카테고리, 브랜드) 기반 최적 크리에이터 Top 10 자동 추천
   - 추천 점수 표시: 인게이지먼트 60% + 매출 실적 40% 가중
   - CJ 캠페인 진행 이력 + 매출 데이터가 있는 크리에이터 가중치 부여
   - "배치 업데이트: YYYY-MM-DD" 표시
   - 각 카드에서 바로 "제안하기" 가능

2. **더보기 (탐색 랜딩)**
   - "더보기" 클릭 시 추천에 사용된 조건(필터)이 세팅된 상태로 T-A-01 크리에이터 탐색으로 이동

3. **노출 영역 (진입점)**
   - 독립 페이지: `/ontrust/creator/recommend`
   - 캠페인 등록/조회 화면 내 추천 영역 (Module로 임베드)
   - 인사이트 리포트(브랜드 기준) 하단 추천 영역

4. **파트너 제한사항**
   - 파트너 계정은 온트너 회원 크리에이터 풀 안에서만 추천 결과 제공

**추천 알고리즘:**
- 매핑 기준: 카테고리 일치도, 브랜드 유사도, 공구 진행 여부
- **랭킹 산정 가중치:**
  - CJ와 캠페인을 진행하여 '매출 데이터' 실적이 있는 크리에이터: 인게이지먼트 60% + 매출 실적 40%
  - 매출 데이터 없는 크리에이터: 인게이지먼트 100%
  - 인게이지먼트 점수는 공통 가중치 공식 적용 (댓글 5.0 > 공유 4.0 > 저장 3.0 > 조회수 2.0 > 좋아요 1.5)
- **데이터 처리: 배치 기반 (1일 1회 업데이트)**
  - 미리 조건에 맞는 풀을 계산해 두는 배치 방식으로 동작 (시스템 과부하 방지)
  - '크리에이터 탐색' 화면 수치와 '추천' 화면 수치가 시간차로 다를 수 있음
  - 배치 업데이트 시각 UI에 표시, 24시간+ 경과 시 경고 배너

**데이터 소스:**
- CJ: 캠페인/매출 데이터
- 피처링: 인게이지먼트 데이터

**미결사항:**
- 추천-탐색 메뉴 구조: [옵션A] 별도 메뉴 vs [옵션B] 탐색 내 추천 탭
- 가중치 비율 최종 확정 필요

#### T-A-15: 컨택 요청 관리 `/ontrust/campaign/contact-requests`
> **유형**: Page | **R&R**: CJ+피처링 협업

**요구사항**: 원시 요구사항 1-3 (크리에이터 탐색 - 컨택 요청) | **상태**: NEW

**목적:**
MD가 크리에이터 탐색(T-A-01) 후 내부 운영 부서(파트너 사업부 등)로 공식 컨택 요청을 보내, 직접 DM을 보내는 실무적 부담을 해소

**컨택 요청 워크플로우:**
1. MD가 T-A-01에서 크리에이터 탐색 → 원하는 크리에이터 선택
2. "컨택 요청" 발송 (캠페인 선택 선택적, 요청 유형/메모 입력)
3. 요청이 T-A-15 목록에 적재
4. 내부 담당자가 상태값으로 처리 상황을 일괄 관리

**주요 기능:**
1. **컨택 요청 목록 테이블**
   - 컬럼: 요청일, 요청자(MD명), 크리에이터(핸들/프로필), 캠페인명, 요청유형, 상태, 처리자, 처리일
   - 필터: 상태별 (대기/처리중/완료/반려), 기간별, 요청자별
   - 정렬: 최신순 (기본), 상태별
   - 페이지네이션 적용

2. **상태 관리**
   - 상태 전이: `대기` → `처리중` → `완료` / `반려`
   - 내부 담당자가 드롭다운으로 상태 변경 (일괄 변경 지원)
   - 상태 변경 시 처리자 ID 및 처리일 자동 기록

3. **요청 유형**
   - `캠페인참여`: 특정 캠페인에 대한 크리에이터 참여 요청
   - `가입제안`: 비회원 크리에이터에 대한 온트너 가입 요청
   - `가입+캠페인`: 가입 제안과 캠페인 참여를 동시에 요청

4. **연결 기능**

## UI/UX 가이드라인

> 참고: 실제 온트러스트는 **웹스퀘어(CJ 내부 프레임워크)** 기반이지만, 본 목업에서는 shadcn/ui로 대체 구현한다. 아래 규칙은 실제 온트러스트 QA 화면의 디자인 패턴을 분석하여 정리한 것이다.

---

### 1. 디자인 원칙

- **플랫 디자인**: 모든 요소의 border-radius 0px (직각 모서리), box-shadow 없음
- **고밀도 레이아웃**: 컴팩트한 입력/버튼/행 사이즈로 한 화면에 많은 정보를 표시
- **전면 보더**: 테이블, 폼 셀 등 모든 영역에 1px solid 보더 적용
- **보라색 포인트**: 주요 액션과 브랜딩에 보라색(#7c3aed) 일관 사용

---

### 2. 레이아웃 구조

#### 2-1. GNB (Global Navigation Bar) — 상단 고정, 2줄

| 줄 | 내용 | 높이 |
|----|------|------|
| 1줄 (메인) | 로고 `ONTRUST` (ON=보라, TRUST=검정) + 카테고리 탭 (고객, 상품, 프로모션, QMS, CAS, 전시, 편성, 채널, 주문, 물류, 통합권, LINK) + 검색 아이콘 + 사용자 아바타 | ~36px |
| 2줄 (보조) | 즐겨찾기, 최근메뉴, 전체메뉴 | 구분선 | 홈 버튼 + 브레드크럼 탭 | ~28px |

- 카테고리 탭 클릭 시 LNB 메뉴가 해당 카테고리로 전환
- 브레드크럼 탭은 최근 방문 페이지를 탭 형태로 표시 (닫기 가능)

#### 2-2. LNB (Left Navigation Bar) — 좌측 사이드바

| 배경 | 흰색 |

- **그룹 헤더**: 좌측 아이콘 + 그룹명 + 우측 펼침/접기 화살표
- **메뉴 아이템**: 들여쓰기 + 점(·) 접두사, 클릭 시 페이지 이동
- **활성 아이템**: 좌측 보라색 보더 + 보라색 텍스트 + 배경 하이라이트
- **비활성 아이템**: 회색 텍스트 + `cursor: not-allowed`
- **NEW 뱃지**: 메뉴 아이템 우측에 빨간색 뱃지
- **하단**: 사용자 정보 카드 (아바타, 이름, 이메일)

- **값 셀** (`otr-form-value`): 흰색 배경, 13px, 좌측 정렬, 1px 보더
- **필수 표시** (`otr-required`): 라벨 뒤에 보라색 `■` 마커
- **값 병합**: `span-2` (3칸), `span-3` (5칸), `full` (나머지 전부)

| 속성 | 규칙 |

| 선택 행 | 연한 보라 배경 (#f3f0ff) |

- **첫 번째 컬럼**: 체크박스 (일괄 선택/개별 선택)
- **두 번째 컬럼**: No (행 번호, 1부터 시작)
- 빈 데이터: 가운데 정렬, "조건에 맞는 데이터가 없습니다." + 아이콘

#### 6-5. 버튼 3종

| 종류 | 클래스 | 배경 | 텍스트 | 보더 | 용도 |

- 예: `[인스타그램] [유튜브]`

#### 6-9. 카테고리 필터 칩 (`otr-classification`)

- 활성: 연보라 배경(#f3f0ff), 보라 보더, 보라 텍스트
- 인라인 나열, 클릭으로 토글

#### 6-10. 페이지네이션

```
[«] [<] [1] [2] [3] [4] [5] [6] [>] [»]  |  총 284건  50건씩 [v]
```

- 활성 페이지: `otr-btn-primary` 스타일 (보라 배경)
- 비활성 페이지: `otr-btn-toolbar` 스타일
- 우측: 총 건수 + 건수/페이지 드롭다운

#### 6-11. 툴바 (`otr-toolbar`)

- 테이블 상단에 배치
- 좌측: 결과 건수, 선택 건수 표시
- 우측: 액션 버튼 (선택승인, 엑셀 다운로드 등)
- 배경: #f7f8fa, 보더: 1px solid #e0e0e0

#### 6-12. 다이얼로그/모달

- 직각 모서리 (border-radius: 0px)
- 헤더: 제목 + 설명
- 본문: space-y-4 간격
- 푸터: 좌측 취소(Secondary) + 우측 확인(Primary)
- 너비: 400~500px

### 7. 페이지 구성 패턴 (3종)

#### 패턴 A: 리스트형 (목록 조회)

```
┌─ 페이지 헤더 (제목 + 설명) ──────────────────────────┐
├─ 검색 패널 (날짜 범위 + 필터 + 조회/초기화) ──────────┤
├─ 툴바 (결과 건수 | 승인/반려 등 액션 버튼) ──────────┤
├─ 데이터 테이블 ──────────────────────────────────────┤
│  □ No | 요청일자 | 요청자ID | ID유형 | 활동명 | ...   │
│  □  1 | 2025.06.20 | ontner02 | ...                  │
│  □  2 | 2025.07.18 | hyshin019 | ...                 │
├─ 페이지네이션 ────────────────────────────────────────┤
└──────────────────────────────────────────────────────┘
```

- 사용처: 크리에이터코드 생성 승인, 캠페인 리스트, 컨택 요청 관리, DM 발송 내역

#### 패턴 B: 폼형 (등록/조회/상세)

```
┌─ 페이지 헤더 (캠페인 코드/명 + 복사/링크생성/조회) ───┐
├─ ◆ 진행 정보 ────────────────────────────────────────┤
│  캠페인 진행 상태 | 모집중    등록자 | 006896 / 장인혜  │
│  등록일시 | 2026-02-11 10:21  수신자 | 006896 / 장인혜  │
├─ ◆ 기본 정보 ────────────────────────────────────────┤
│  캠페인코드 | CP00001034    캠페인명 | ...              │
│  캠페인 유형 | [광고제휴(기간모집형) v]                   │
│  지급액 | 100,000 원        지급율 | %                  │
│  모집 대상등급 | [퍼플] [민트] [그린] [블루] [블랙]       │
├─ ◆ 상세 정보 (필요 시) ──────────────────────────────┤
└──────────────────────────────────────────────────────┘
```

- 사용처: 캠페인 등록/조회, 크리에이터 상세, DM 어드민 설정
- 섹션 마커(◆)로 영역 구분
- 폼 그리드(2~4열)로 라벨-값 배치

#### 패턴 C: 대시보드형 (통계 + 목록)

```
┌─ 통계 카드 (5열 그리드) ─────────────────────────────┐
│  [전체 42] [대기 12] [처리중 8] [완료 18] [반려 4]     │
├─ 필터 바 (상태 드롭다운 + 검색 + 일괄 액션) ──────────┤
├─ 데이터 테이블 ──────────────────────────────────────┤
├─ 페이지네이션 ────────────────────────────────────────┤
└──────────────────────────────────────────────────────┘
```

- 사용처: 컨택 요청 관리, DM 성과 대시보드
- 통계 카드: 상태별 색상 (회색=전체, 노랑=대기, 파랑=처리중, 초록=완료, 빨강=반려)

### 8. 인터랙션 규칙

| 인터랙션 | 동작 |
|----------|------|
| 행 선택 | 체크박스 클릭 → 연보라 배경(#f3f0ff) 하이라이트 |
| 행 호버 액션 | 호버 시 우측에 `⋯` 더보기 메뉴 표시 (opacity 전환) |
| 비활성 메뉴 | 회색 텍스트 + `cursor: not-allowed` + 클릭 불가 |
| 필수 필드 | 라벨 뒤에 보라색 `■` 마커 |
| 비활성 필터 | UI에 표시하되 비활성 상태 + 툴팁 "데이터 미제공" |
| 드롭다운 상태 | 전체/요청완료/반려/승인완료 등 상태 옵션 |
| 일괄 액션 | N건 선택 시 툴바에 일괄 액션 버튼 활성화 |
| 링크 클릭 | 텍스트 밑줄 + 보라색, 클릭 시 상세 페이지 이동 |

### 9. CJ 자체 영역 vs 피처링 구현 영역

- **CJ 자체 메뉴** (disabled): LNB에 표시하되 클릭 불가, 회색 처리
  - 예: 온트너 계정 관리, 캠페인 리스트, 정산현황, 전자계약, 고객, 상품 일부
- **피처링 구현 영역**: 활성 상태, 클릭 가능, 보라색 하이라이트
  - 예: 크리에이터 탐색, 인사이트 리포트, DM 관리, 성과 조회
- CJ 자체 영역의 페이지 내용은 구현하지 않되, 사이드바 메뉴에는 표시하여 실제 시스템의 전체 구조를 보여준다
